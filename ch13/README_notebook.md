# 13주차: 통합 정책 시뮬레이션 워크숍

## 수업 목표

이번 주차는 새 도구를 배우지 않는다. 한 학기 동안 익힌 5개 도구(인과 분석·시스템 사고·베이지안·시나리오·몬테카를로)를 **하나의 정책 사례**(청년 주거 지원금)에 차례대로 적용한다.

학생은 수업 후 다음을 할 수 있어야 한다.

1. 한 정책 사례에 5개 도구를 적용해 5가지 인사이트를 얻을 수 있다.
2. 도구별 결과가 보완·충돌하는 지점을 설명할 수 있다.
3. 다섯 결과를 하나의 권고안으로 종합할 수 있다.
4. 통합 기획서의 골격을 워크숍 결과로 채울 수 있다.

## 핵심 개념

| 단계 | 도구 | 묻는 질문 | 활용 주차 |
|------|------|---------|---------|
| 1 | DAG | 무엇이 무엇을 일으키는가? | 4주차 |
| 2 | CLD·스톡-플로우 | 피드백과 시간 지연은? | 5주차 |
| 3 | 베이지안 | 새 정보로 믿음을 어떻게 갱신? | 6주차 |
| 4 | 시나리오 | 어떤 미래에서도 살아남나? | 7주차 |
| 5 | 몬테카를로 | 결과 분포와 달성 확률은? | 9주차 |

## 진행 방식

1. 0:00-0:10 — 사례 소개 (청년 주거 1,000억)
2. 0:10-1:00 — 5개 도구 순차 적용 (각 10분)
3. 1:00-1:30 — 통합 인사이트 도출 (충돌 지점 식별)
4. 1:30-3:00 — 학생 실습 (자기 주제로 5단계 반복)

## 바이브 코딩 방식

각 단계마다 AI에게 보낼 요청 1개가 노트북에 명시되어 있다. 학생은 AI 응답을 빈 셀에 붙여 넣고 실행한 뒤, 결과가 다른 도구의 결과와 어떻게 충돌·보완하는지 한 줄로 메모한다.

## 학기 통합

이번 워크숍의 결과는 다음과 직접 연결된다.

- **14주차**: 워크숍 결과를 발표하고 동료 피드백을 받는다.
- **15주차 기말고사**: 5개 도구의 통합 적용을 묻는 사례 분석 문항.
- **기말 프로젝트(통합 기획서)**: 워크숍 결과가 그대로 골격이 된다.
