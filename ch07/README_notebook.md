# 10주차: 시나리오 플래닝

## 수업 목표

이번 주차의 목표는 미래를 정확히 맞히는 것이 아니라, **여러 가능한 미래에 대비하는 사고방식**을 익히는 것이다.

학생은 수업 후 다음을 할 수 있어야 한다.

1. 예측(forecasting)과 시나리오 플래닝(scenario planning)의 차이를 설명할 수 있다.
2. 동인(driving forces)을 식별하고 STEEP 프레임워크로 분류할 수 있다.
3. 불확실성-영향도 매트릭스로 핵심 불확실성을 골라낼 수 있다.
4. 2×2 매트릭스로 4개 시나리오 서사를 만들 수 있다.
5. 윈드 터널링으로 전략의 강건성을 평가할 수 있다.
6. 시나리오마다 조기 경보 지표를 설계할 수 있다.

## 핵심 개념

### 예측 vs 시나리오 플래닝

| 구분 | 예측 (Forecasting) | 시나리오 플래닝 |
|------|------------------|----------------|
| 목적 | 가장 가능성 높은 미래 1개 | 가능한 미래 3-4개 |
| 가정 | 추세가 이어진다 | 추세가 끊길 수 있다 |
| 결과물 | 숫자 하나 | 서사 여러 개 |
| 적합 상황 | 안정적 환경 | 격변기·구조 변화 |

### 동인 (Driving Forces)

미래를 결정하는 큰 흐름. STEEP 프레임워크로 5개 영역으로 분류한다.

- **S**ocial — 사회·문화·인구
- **T**echnological — 기술
- **E**conomic — 경제
- **E**cological — 환경·기후
- **P**olitical — 정치·규제

### 불확실성-영향도 매트릭스

동인 중 **불확실하면서 영향도가 큰 것**이 핵심 불확실성이다. 그 중 **서로 독립적인 두 가지**가 시나리오의 두 축이 된다.

```
영향도 높음
   |
   |  결정 요인         ★ 핵심 불확실성
   |  (이미 정해진 것)    (시나리오 축으로 사용)
   |
   +-------------+-------------
   |
   |  무시 가능         관심사
   |                   (배경 변수)
   |
   +-------------------- 불확실성 높음
```

### 2×2 시나리오 매트릭스 매뉴얼

1. 두 축을 정한다.
2. 각 축에서 **두 극단**을 정한다 (예: 기술 빠름 ↔ 느림).
3. 4개 사분면에 시나리오 이름을 붙인다.
4. 각 시나리오에 **서사**를 쓴다(2~3 문단). 등장인물, 사건, 결과를 포함한다.
5. 각 시나리오에 **확률**을 매기지 않는다 — 시나리오는 가능성이지 예측이 아니다.

### 윈드 터널링 (Wind Tunneling)

전략 후보를 4개 시나리오에 모두 통과시켜 본다. 어느 시나리오에서도 살아남는 전략이 **강건한(robust) 전략**이다.

| 전략 / 시나리오 | S1 | S2 | S3 | S4 | 강건성 |
|----------------|----|----|----|----|------|
| 전략 A | + | + | - | + | 3승 1패 |
| 전략 B | ++ | -- | -- | ++ | 양극 |
| 전략 C | + | + | + | + | 모두 양호 ★ |

### 조기 경보 지표 (Early Warning Indicators)

각 시나리오가 현실이 되어 가는지 미리 알려주는 지표. 시나리오마다 **2~3개**를 정한다.

- 측정 가능해야 한다.
- 분기·반기 단위로 모니터링 가능해야 한다.
- 임계값을 정해 둔다 ("X가 Y를 넘으면 시나리오 S2로 이동").

## 바이브 코딩 방식

학생은 코드를 처음부터 외워서 작성하지 않는다.

1. AI에게 동인 12개를 STEEP으로 분류해 달라고 요청한다.
2. 각 동인에 불확실성·영향도 점수(0-10)를 매긴다.
3. AI에게 매트릭스 시각화 코드를 요청한다.
4. 두 축을 정하고 4시나리오 이름을 짓는다.
5. AI에게 시나리오별 서사를 요청해 다듬는다.

예시 프롬프트:

```text
나는 [주제, 예: 2030년 한국 도시 모빌리티]에 대한 시나리오 플래닝을 하려고 해.
STEEP 프레임워크로 동인 12개를 만들어주고, 각 동인에 불확실성과 영향도 점수(0-10)를 매겨줘.
표 형식으로 정리해줘.
```
