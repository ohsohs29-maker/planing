# 11주차: 성과 측정과 KPI

## 수업 목표

이번 주차의 목표는 통계 기법을 배우는 것이 아니라, **무엇을 측정하면 정책·사업이 잘 굴러가는지 알 수 있는가**를 정하고, 이상 신호를 빠르게 잡아내는 사고방식을 익히는 것이다.

학생은 수업 후 다음을 할 수 있어야 한다.

1. 성과 측정의 목적을 한 문장으로 설명할 수 있다.
2. 선행 지표와 후행 지표를 구분할 수 있다.
3. SMART 원칙으로 KPI를 작성할 수 있다.
4. Balanced Scorecard 4관점으로 KPI를 균형 있게 배치할 수 있다.
5. 이상 패턴을 직관적으로 탐지할 수 있다.
6. KPI 대시보드 초안을 그릴 수 있다.

## 핵심 개념

### 성과 측정의 본질

피터 드러커: *"측정되지 않으면 관리되지 않는다."*

기획서에 KPI가 없으면 실행 단계에서 진행 상황을 알 수 없다. 그러나 **잘못 측정하면 잘못된 행동을 부른다**. 측정 지표가 행동을 결정하기 때문이다.

### 선행 지표 vs 후행 지표

| 구분 | 선행 지표 (Leading) | 후행 지표 (Lagging) |
|------|-------------------|--------------------|
| 시점 | 결과 전 | 결과 후 |
| 통제 가능성 | 통제 가능 | 통제 불가 |
| 예시(다이어트) | 일일 칼로리 섭취 | 체중 |
| 예시(매출) | 영업 미팅 수 | 분기 매출 |
| 예시(정책) | 시범 신청 건수 | 1년 후 고용률 |

> **기획자의 질문**: 우리 KPI가 모두 후행이면, 손쓸 시점에는 이미 늦는다. 선행 지표를 함께 두자.

### SMART 매뉴얼

KPI를 작성할 때 5가지를 점검한다.

| 글자 | 점검 질문 |
|------|---------|
| **S**pecific | 무엇을 측정하는지 한 문장으로 적을 수 있는가 |
| **M**easurable | 숫자로 측정 가능한가 |
| **A**chievable | 달성 가능한 수준인가 (너무 쉽지도 않은가) |
| **R**elevant | 핵심 목표와 연결되는가 |
| **T**ime-bound | 언제까지인가 |

### Balanced Scorecard 4관점

KPI를 한 영역에 몰면 균형이 깨진다. 4관점에 골고루 둔다.

| 관점 | 핵심 질문 | 예시 KPI |
|------|---------|---------|
| 재무 | 주주·예산은 만족하는가 | 순이익, ROI |
| 고객 | 시민·고객은 만족하는가 | NPS, 재이용률 |
| 내부 프로세스 | 내부 운영은 효율적인가 | 처리 시간, 오류율 |
| 학습·성장 | 조직은 미래에 준비되어 있는가 | 직원 교육 시간, 신기술 도입 |

### 이상 탐지의 직관

복잡한 알고리즘 없이도 다음 두 방법으로 충분히 잡을 수 있다.

1. **이동 평균 (Moving Average)** — 단기 흐름이 장기 흐름과 크게 어긋나면 이상.
2. **표준편차 밴드 (±2σ)** — 평균에서 표준편차 2배를 벗어나면 이상 의심.

```
이상 의심  =  현재 값  >  이동평균 + 2 * 이동표준편차
```

### KPI 대시보드 매뉴얼

1. 핵심 KPI **5~7개**를 고른다 (너무 많으면 안 본다).
2. 각 KPI에 **목표값**과 **임계값**을 설정한다.
3. 색상 대신 **텍스트로 상태**를 표시한다 (Good / Watch / Alert).
4. 한 화면에 다 보이도록 한다.

## 바이브 코딩 방식

학생은 코드를 외우지 않는다.

1. 정책·사업 목표를 적는다.
2. AI에게 선행/후행 KPI를 6~8개 추천받는다.
3. SMART 원칙으로 KPI를 다듬는다.
4. AI에게 시계열 데이터와 이상 탐지 코드를 요청한다.
5. 대시보드 형태로 정리한다.

예시 프롬프트:

```text
나는 [정책/사업명]의 KPI 체계를 설계하려고 해.
선행 지표 4개와 후행 지표 4개를 추천해주고, 각각을 SMART 형식으로 작성해줘.
Balanced Scorecard 4관점으로 분류도 해줘.
표 형식으로 정리해줘.
```
