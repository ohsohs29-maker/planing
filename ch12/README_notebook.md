## 핵심 개념

### 전략-실행 갭이 생기는 이유

좋은 전략 슬라이드가 잘 만들어졌는데도 결과가 안 나오는 5가지 패턴:

1. 전략이 너무 추상적이어서 누가 무엇을 할지 모른다.
2. 책임이 흩어져 있어 결정이 미뤄진다.
3. 리스크가 미리 식별되지 않아 사고가 나면 뒤늦게 대응한다.
4. KPI가 없어 진행 상황을 모른다 (13주차 연결).
5. 환경 변화에 계획이 따라가지 못한다 (14주차 연결).

12주차는 1, 2, 3을 다룬다.

### OKR vs SMART

| 도구 | 강점 | 한계 | 적합 상황 |
|------|------|------|---------|
| **OKR** (Objectives & Key Results) | 야심찬 목표·정렬 | 분기 단위, 기업 문화 필요 | 분기 목표·신사업 |
| **SMART** | 구체성·검증성 | 야심·정렬 약함 | 개인·팀 단위 운영 KPI |

**OKR 매뉴얼 (3단계)**

1. **Objective** 한 문장: 정성적·야심적·기억 가능 (예: "지역 청년의 첫 직장 진입을 빠르고 쉽게 만든다").
2. **Key Results** 3~5개: 측정 가능한 결과 (예: "Q2 신청 200건", "채용 전환율 65%").
3. **목표 달성도 0.6~0.7이면 성공** — 1.0이면 너무 쉬운 목표.

**SMART 매뉴얼**

13주차 KPI 매뉴얼과 동일.

### WBS (Work Breakdown Structure)

큰 일을 작은 일로 쪼갠다. **3 레벨, 8주 단위**가 가독성이 좋다.

```
Level 1: 정책 시범사업 운영 (전체)
  Level 2: 모집 / 교육 / 매칭 / 평가
    Level 3 (모집): 홍보 / 신청 시스템 / 서류 검토 / 면담
```

각 Level 3 작업에 **기간(주)**, **담당자**, **산출물**을 적는다.

### RACI 매트릭스

각 작업에 4역할을 배정한다.

| 글자 | 의미 | 인원 |
|------|------|------|
| **R**esponsible | 직접 일하는 사람 | 1명 이상 |
| **A**ccountable | 결정·승인하는 사람 | **반드시 1명** |
| **C**onsulted | 의견을 듣는 사람 | 0명 이상 |
| **I**nformed | 결과만 통보받는 사람 | 0명 이상 |

> **한 작업에 A가 2명 이상이면 결정이 안 된다.**

### 리스크 식별과 매트릭스

리스크 = 발생 확률 × 영향도. 매트릭스에 점을 찍어 우선순위를 본다.

```
영향도 높음
   |
   |  주의 관찰      ★ 즉시 대응
   |  (Watch)         (Act now)
   |
   +-------------+-------------
   |
   |  무시 가능      완화 검토
   |                  (Mitigate)
   |
   +-------------------- 발생 확률 높음
```

### 4대 대응 전략

| 전략 | 의미 | 예시 |
|------|------|------|
| **회피 (Avoid)** | 리스크 원인 자체 제거 | 위험 지역 사업 철수 |
| **완화 (Mitigate)** | 확률·영향 감소 | 백업, 다중 공급선, 사전 점검 |
| **전가 (Transfer)** | 제3자에게 이전 | 보험, 외주, 계약 조항 |
| **수용 (Accept)** | 그대로 받음 | 영향 작은 위험은 모니터만 |

**선택 매뉴얼**

```
영향 큼 + 확률 높음 → 회피 우선, 안 되면 완화
영향 큼 + 확률 낮음 → 전가 (보험 적합)
영향 작음 + 확률 높음 → 완화
영향 작음 + 확률 낮음 → 수용 (모니터링)
```

### 리스크 레지스터

각 리스크를 1행으로 정리한다.

| 컬럼 | 내용 |
|------|------|
| ID | R1, R2... |
| 설명 | 한 문장 |
| 분류 | 기술/일정/비용/자원/외부/조직 |
| 확률 | 0~1 |
| 영향 | 1~5 |
| 점수 | 확률 × 영향 |
| 전략 | 회피/완화/전가/수용 |
| 대응안 | 구체 행동 |
| 담당자 | 1명 |
| 트리거 | 조기 경보 지표 |

### 조기 경보 지표

리스크가 현실이 되기 전에 발견할 수 있는 지표. 13주차 KPI와 14주차 적응적 트리거에 그대로 연결된다.

## 바이브 코딩 방식

학생은 코드를 외우지 않는다.

1. 정책·사업 목표를 한 문장으로 적는다.
2. AI에게 OKR 1개와 KR 3~5개를 추천받는다.
3. AI에게 WBS Level 2~3을 받는다.
4. AI에게 리스크 8~10개를 받는다 (분류·확률·영향 포함).
5. AI에게 매트릭스 시각화 코드와 대응 전략을 요청한다.

예시 프롬프트:

```text
나는 [정책/사업명] 의 실행 계획을 세우려고 해.
1) Objective 한 문장과 Key Results 3~5개를 만들어줘.
2) WBS를 Level 2(4-6개), Level 3(각 3-4개)로 정리해줘.
3) 주요 리스크 8개를 표로 만들고 분류·확률(0-1)·영향(1-5)·점수를 계산해줘.
4) 각 리스크에 회피/완화/전가/수용 중 적절한 전략과 대응안을 제안해줘.
```
