# 14주차: 적응적 기획과 기말 프로젝트 정리

## 수업 목표

이번 주차의 목표는 **계획을 한 번 세우고 끝내는 사고**를 버리고, 환경이 바뀔 때 계획을 다시 짜는 **롤링 플래닝**을 익히는 것이다. 동시에 한 학기 동안의 학습을 **통합 기획서**로 묶는다.

학생은 수업 후 다음을 할 수 있어야 한다.

1. 전통적 기획과 적응적 기획의 차이를 한 문장으로 설명할 수 있다.
2. 롤링 플래닝의 주기와 의미를 설명할 수 있다.
3. 계획 수정 트리거를 설계할 수 있다.
4. AAR(After Action Review)로 학습을 정리할 수 있다.
5. 한 학기 학습을 통합 기획서로 정리할 수 있다.

## 핵심 개념

### 전통적 기획 vs 적응적 기획

| 구분 | 전통적 기획 | 적응적 기획 |
|------|-----------|-----------|
| 가정 | 환경이 안정 | 환경이 변한다 |
| 주기 | 1년에 한 번 | 분기·반기마다 갱신 |
| 변경 | 어렵다 | 트리거에 따라 자동 |
| 학습 | 사후 평가 | 진행 중 학습 |
| 적합 상황 | 정규 운영 | 신사업·신정책·격변기 |

### 적응적 사이클 (Plan-Do-Check-Adapt)

```
   Plan (계획)
      ↓
   Do (실행)
      ↓
   Check (모니터링) ← 13주차 KPI
      ↓
   Adapt (계획 수정) ← 트리거 발동 시
      ↓
   (다시 Plan)
```

핵심은 **Adapt 단계가 정해진 시점에 정해진 트리거로 자동 발동**되어야 한다는 점이다. 아니면 \"항상 계획대로\"가 된다.

### 롤링 플래닝 매뉴얼

전통적 기획은 1년 계획을 세워 1년 동안 그대로 간다. 롤링 플래닝은 다음 구조를 쓴다.

| 기간 | 상세도 | 갱신 주기 |
|------|------|---------|
| 다음 분기 | 매우 상세 (주별 일정) | 월 |
| 6개월~1년 | 보통 (월별) | 분기 |
| 1~3년 | 개략 (분기별) | 반기 |
| 3~5년 | 비전·방향만 | 연 |

**작성 단계 (5단계)**

1. 시간 지평 4개 구간을 정한다.
2. 구간마다 상세도를 정한다.
3. 각 구간의 갱신 주기를 달력에 못 박는다.
4. 갱신 시 가장 가까운 구간만 다시 짜고, 먼 구간은 비전만 점검한다.
5. 계획 수정 이력을 한 페이지로 남긴다 (왜 바뀌었는가).

### 계획 수정 트리거

미리 정해 두는 \"이게 발생하면 계획을 다시 본다\" 신호. 13주차 KPI 시스템과 직접 연결된다.

| 트리거 유형 | 예시 |
|-----------|------|
| KPI 임계값 위반 | 분기 매출 -20% 또는 신청 건수 -30% |
| 외부 이벤트 | 신규 규제 발표, 경쟁사 인수합병 |
| 정해진 일정 | 분기 종료, 시범사업 종료 |
| 가설 검증 결과 | 베이지안 사후확률이 임계 이하로 떨어짐 |

**매뉴얼**

1. 핵심 가정 5개 적기 (\"X가 Y이면 계획이 통한다\").
2. 각 가정마다 트리거 1~2개 적기 (\"X가 Z를 넘으면 가정 깨짐\").
3. 트리거 발동 시 누가 무엇을 결정하는지 적기 (RACI 적용).
4. 트리거를 KPI 대시보드에 표시.

### After Action Review (AAR)

미군에서 시작된 학습 도구. 4개 질문으로 빠르게 점검한다.

1. 무엇이 일어나기를 기대했는가?
2. 실제로 무엇이 일어났는가?
3. 왜 차이가 났는가?
4. 다음에 무엇을 다르게 할 것인가?

회의는 30분 이내, 비난 금지, 발견 사항은 문서화하고 공유.

### 통합 기획서 구조 (기말 프로젝트)

한 학기 동안 배운 도구를 모두 한 주제에 적용한 결과물.

| 섹션 | 활용 주차 |
|------|---------|
| 1. 문제 정의 (MECE·로직 트리) | 3주차 |
| 2. 인과 분석 (DAG) | 4주차 |
| 3. 시스템 사고 (CLD) | 5주차 |
| 4. 환경 스캐닝과 약신호 | 6주차 |
| 5. 이해관계자 분석 | 7주차 |
| 6. 베이지안 의사결정 | 9주차 |
| 7. 시나리오 플래닝 | 10주차 |
| 8. 몬테카를로 시뮬레이션 | 11주차 |
| 9. 실행 계획과 리스크 | 12주차 |
| 10. KPI와 모니터링 | 13주차 |
| 11. 적응적 기획 (트리거·AAR) | 14주차 |

## 바이브 코딩 방식

이번 주차는 코드 비중이 작다. 트리거 점검과 AAR 보고서 자동화 정도만 사용한다.

예시 프롬프트:

```text
내 정책의 핵심 가정 5개와 KPI 트리거 5개를 표로 정리해줘.
각 트리거마다 발동 조건과 책임자(RACI), 다음 행동을 적어줘.
```

```text
이번 분기 KPI 데이터와 목표를 줄게.
AAR 4개 질문에 답하는 양식을 만들고, 데이터를 바탕으로 1번과 2번을 자동 채워줘.
```
