# 9주차: 쉬운 베이지안 의사결정

## 수업 목표

이번 주차의 목표는 베이즈 정리 수학 증명이 아니라, 기획자가 **새로운 정보로 믿음을 갱신하고** 의사결정을 바꾸는 사고방식을 익히는 것이다.

학생은 수업 후 다음을 할 수 있어야 한다.

1. 점 추정의 한계를 한 문장으로 설명할 수 있다.
2. 사전확률, 가능도, 사후확률을 일상 예시로 구분할 수 있다.
3. 새로운 정보가 들어왔을 때 사후확률을 계산할 수 있다.
4. AI에게 베이지안 업데이트 코드를 요청하고 결과를 해석할 수 있다.

## 핵심 개념

### 점 추정의 한계

"이 사업의 매출은 1억"처럼 하나의 숫자로 표현하면 **불확실성이 사라진다**. 실제로는 "8천만에서 1.2억 사이일 가능성이 높다"가 정확하다. 베이지안은 이 분포를 그대로 다룬다.

### 사전확률 / 가능도 / 사후확률

```
사후확률(Posterior)  =  가능도(Likelihood)  ×  사전확률(Prior)
                      ─────────────────────────────────────
                                 정규화 상수
```

쉽게 말해:

- **사전확률**: 정보를 보기 전 내가 가진 믿음
- **가능도**: 실제 데이터가 이 가설 아래에서 얼마나 그럴듯한가
- **사후확률**: 데이터를 본 다음의 갱신된 믿음

### 베이지안 업데이트의 직관

데이터가 적을 때는 **사전확률**의 영향이 크다. 데이터가 쌓이면 **사후확률은 데이터에 가까워진다**. "결국 데이터가 이긴다."

### 기획자의 질문

- 지금 가진 믿음의 근거는 무엇인가?
- 어떤 정보가 들어오면 결정을 바꿀 것인가?
- 추가 조사를 할 가치가 있는가?

## 바이브 코딩 방식

학생은 코드를 처음부터 외워서 작성하지 않는다. 대신 아래 순서로 진행한다.

1. 분석하고 싶은 일을 자연어로 쓴다.
2. AI에게 Python 코드를 요청한다.
3. 코드를 실행한다.
4. 오류가 나면 오류 메시지를 AI에게 다시 보여준다.
5. 결과를 표와 짧은 문장으로 해석한다.

예시 프롬프트:

```text
어떤 정책의 성공 확률에 대해 처음에는 30%라고 믿고 있었어.
시범 사업 10건 중 7건이 성공했어.
이 정보로 사후확률을 계산하는 Python 코드를 만들어줘.
사전·가능도·사후 분포를 같은 그래프에 흑백으로 그려줘.
```
